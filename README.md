# Fake Loop - 蓝牙跳绳模拟器

仅供学习交流使用。

将手机模拟成一个蓝牙跳绳设备（Loop 跳绳），通过标准 BLE 外设（Peripheral）模式广播，供第三方 App 连接并接收跳绳数据。

## 功能

- 以设备名 `LR029429BD` 自动启动 BLE 外设广播。
- 支持手动发送开始包、800 下、1600 下和自定义 Hex 数据。
- 支持自动跳绳：可按目标时间或平均速度运行，并设置目标跳数、减速率和随机跳数。
- 自动模式会显示完成进度、当前速度、用时和剩余时间。
- 自动模式支持暂停、继续和停止。
- 自动模式可选择保持屏幕常亮，运行结束或停止后自动恢复。
- 目标跳数、目标时间、减速率、随机开关和防熄屏开关会保存到本机偏好设置。

## BLE 服务与特征

以设备名 `LR029429BD` 广播，暴露自定义服务 `FFF0`，包含两个特征：

- `FFF1`：通知特征（Notify），App 通过此特征发送跳绳数据包。
- `FFF2`：读写特征（Read/Write），保留给外部设备写入配置等（当前未使用）。

### 广播包结构

广播包包含以下段：

- 设备名 `0x09`：`4C 52 30 32 39 34 32 39 42 44`（ASCII `LR029429BD`）
- 制造商数据 `0xFF`：公司 ID `0x00EE`，负载 `4C 4F 4F 50 00 02 00`（ASCII `LOOP` + `00 02 00`）
- 16 位服务 UUID `0xFFF0`

### GATT 服务器

| 角色 | UUID（128-bit） | 属性 | 权限 |
| --- | --- | --- | --- |
| 服务 | `0000fff0-0000-1000-8000-00805f9b34fb` | Primary | - |
| 通知特征 | `0000fff1-0000-1000-8000-00805f9b34fb` | Notify + Read | Read |
| 读写特征 | `0000fff2-0000-1000-8000-00805f9b34fb` | Read + Write | Read + Write |
| CCCD | `00002902-0000-1000-8000-00805f9b34fb` | - | Read + Write |

## 数据包格式

开始包固定为 6 字节：

```text
6F 02 01 00 00 72
```

跳绳数据包固定为 16 字节：

```text
6F 04 0B 00 00 00 XX XX 69 .. .. .. .. .. .. CS
```

| 偏移 | 字节数 | 内容 | 说明 |
| --- | --- | --- | --- |
| 0 | 1 | `6F` | 数据包头标识 |
| 1-2 | 2 | `04 0B` | 数据包类型标识 |
| 3-5 | 3 | `00 00 00` | 保留填充位 |
| 6-7 | 2 | `XX XX` | 跳数乘以 10，big-endian |
| 8 | 1 | `69` | 固定标识 |
| 9-14 | 6 | 插值字段 | 基于 800 和 1600 跳参考包线性插值 |
| 15 | 1 | `CS` | 前 15 字节求和后取低 8 位 |

## 自动跳绳逻辑

自动模式支持两种运行方式：

- 按时间：使用目标跳数和目标时间计算平均速度。
- 按速度：使用目标跳数和平均速度反推预计用时。

两种方式都会按减速率生成每秒速度。减速率为 `0` 时接近匀速，越接近 `1` 前快后慢越明显。启用随机跳数后，本次运行会在目标跳数基础上随机增加 `0~50` 下，并在运行状态中显示本次实际增加值。

跳数使用 2 字节保存“跳数乘以 10”的值，因此自动模式会限制目标跳数，避免数据包溢出。
