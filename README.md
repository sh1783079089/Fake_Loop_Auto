## BLE 服务与特征

以设备名 `LR029429BD` 广播，暴露自定义服务 `FFF0`，包含两个特征：
- `FFF1`：通知特征（Notify），App 每秒发送一个跳绳数据包到此特征，供已订阅设备接收。
- `FFF2`：读写特征（Read/Write），保留给外部设备写入配置等（当前未使用）。
- 
### 广播包结构

广播包包含以下段：

- 设备名 `0x09`：`4C 52 30 32 39 34 32 39 42 44`（ASCII `LR029429BD`）
- 制造商数据 `0xFF`：公司 ID `0x00EE`，负载 `4C 4F 4F 50 00 02 00`（ASCII `LOOP` + 0x00 0x02 0x00）
- 16 位服务 UUID `0xFFF0`

### GATT 服务器

| 角色 | UUID（128-bit） |属性 | 权限 |
|------|----------------|------|------|
| 服务 | `FFF0`（0000fff0-0000-1000-8000-00805f9b34fb） | Primary | — |
| 通知特征 | `FFF1`（0000fff1-0000-1000-8000-00805f9b34fb） | Notify + Read | Read |
| 读写特征 | `FFF2`（0000fff2-0000-1000-8000-00805f9b34fb） | Read + Write | Read + Write |
| 客户端特征配置描述符（CCCD） | `00002902-0000-1000-8000-00805f9b34fb` | — | Read + Write |

- **通知特征**负责发送跳绳数据：外部设备订阅此特征后，App 每秒发送一次数据包（见下节《数据包格式》）。
- **读写特征**当前未使用。

## 数据包格式

跳绳数据包为固定 18 字节的十六进制字符串，格式如下：

```
6F 04 0B 00 00 00 XXXX 6930 0057 03D8 02A8
```

| 偏移 | 字节数 | 内容 | 说明 |
|------|--------|------|------|
| 0 | 1 | `6F` | 数据包头标识 |
| 1–2 | 2 | `04 0B` | 数据包类型标识（固定 `04 0B`） |
| 3–5 | 3 | `00 00 00` | 保留填充位 |
| 6–7 | 2 | `XXXX` | **跳数 × 10 的 4 位十六进制大写**（例如 800 跳 → `1F40`） |
| 8–9 | 2 | `6930` | 数据尾标识 |
| 10–11 | 2 | `0057` | 保留填充 |
| 12–13 | 2 | `03D8` | 保留填充 |
| 14–15 | 2 | `02A8` | 保留填充 |

**开始包**：当用户点击「开始跳绳」后，发送：

```
6F 02 01 00 00 72
```

- 该包固定为 6 字节，标识跳绳开始，不含跳数数据。
